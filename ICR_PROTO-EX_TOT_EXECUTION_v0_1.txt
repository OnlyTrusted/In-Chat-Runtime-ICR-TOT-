STATE HEADER : [STATE: EXECUTING | PROTO: PROTO-EX | SEQ: 1/4 | SYNC: <status>]

EXECUTION:
  1. Parse task — extract goal, constraints, requirements
  2. Generate 3 distinct branches:
     Each branch contains:
       • BRANCH ID    : A | B | C
       • APPROACH     : what strategic angle this takes
       • LOGIC        : how it solves the task step by step
       • REQUIREMENTS : what it needs to execute
       • TRADEOFFS    : what it gains and what it costs
       • BEST FOR     : what conditions make this branch optimal
  3. Branches must be meaningfully distinct —
     not variations of the same approach

    BEST FOR     : ...

STATE HEADER : [STATE: EXECUTING | PROTO: PROTO-EX | SEQ: 3/4 | SYNC: <status>]

GRADING AXES (applied to each branch):
  • FEASIBILITY      : can it actually be executed given constraints
  • GOAL ALIGNMENT   : how well it satisfies the task goal
  • COMPLEXITY       : execution cost and overhead
  • CONSTRAINT FIT   : adherence to stated requirements
  • OS MATCH         : compatibility with ICR runtime and format rules

GRADING SCALE: 1-5 per axis

EXECUTION:
  1. Grade each branch on all 5 axes
  2. Dissect strengths and weaknesses per branch
  3. Identify standout features per branch
     (things the other branches don't have)
  4. Select recommended branch based on highest
     weighted score — GOAL ALIGNMENT and OS MATCH
     weighted x2
  5. Surface options to user

OUTPUT FORMAT:

  ── BRANCH A — <name> ──────────────────────────────
  FEASIBILITY    : [n/5] — <reason>
  GOAL ALIGNMENT : [n/5] — <reason>

  CONSTRAINT FIT : [n/5] — <reason>
  OS MATCH       : [n/5] — <reason>
  TOTAL          : [weighted score]
  STANDOUT       : <what only this branch has>
  VERDICT        : <1-line summary>

       • Assess execution cost of adding it

       • DISCARD if: conflicts or high cost
  4. Output refined solution + recommendations
  5. Close unit — emit completion signal to audit
